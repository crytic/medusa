# Call sequences
A call sequence is a series of contract calls. The calls can be put into different blocks at different timestamps; this is represented by `CallSequenceElement.BlockNumberDelay` and `CallSequenceElement.BlockTimestampDelay`. `ExecuteCallSequenceIteratively` handles executing call sequences. Most of the logic has to do with handling blocks and block numbers/timestamps. For example, when `CallSequenceElement.BlockNumberDelay` is set, we can often overwrite the current block's number rather than actually creating a new block, in order to save execution time.
`func ExecuteCallSequenceIteratively(chain *chain.TestChain, fetchElementFunc ExecuteCallSequenceFetchElementFunc, executionCheckFunc ExecuteCallSequenceExecutionCheckFunc, additionalTracers ...*chain.TestChainTracer) (CallSequence, error)` takes a chain, a function to get call sequence elements, a function run after each call sequence is executed, and tracers to instrument execution. It returns the call sequence that was run. This returned `CallSequence` includes "receipts" that describe what block it was in, and what the block number and block timestamp were at the time of execution. We record the block number and block timestamp in the receipt since this information can't reliably be grabbed from the block afterwards, since we sometimes overwrite these values during `ExecuteCallSequenceIteratively`.
`ExecuteCallSequence` and `ExecuteCallSequenceWithExecutionTracer` are helper functions that wrap `ExecuteCallSequenceIteratively`.
